# Le Chef — анализ конкурентов (стартовый, июнь 2026)

## Сводная таблица

|---|---|---|---|---|
| SuperCook | Рецепты из имеющихся продуктов | Галочки ингредиентов, десятки тыс. рецептов, «чего не хватает» | Free + реклама | Ввод вручную, нет фото-скана |

## Главный вывод
Рынок делится на два лагеря, и почти никто не соединяет их в одном продукте:
- Серьёзные утилиты: фото -> ингредиенты -> рецепты -> список покупок.
- Весёлые решалки: колесо/квиз «что/где поесть».

Пробел, который закрывает Le Chef: единый продукт, где fun-вход (колесо/квиз)
ведёт к серьёзному ядру (скан -> рецепты -> покупки -> бронь шефа/ресторана).
Плюс почти ни у кого нет связки «домашняя готовка <-> рестораны <-> шеф».

## Что подтверждает гипотезу «от fun к серьёзному»
- Food-Decider: «no sign-up» + колесо дали мгновенную вовлечённость.
- SpinDine: геймификация работает только если за ней стоит реальная система;
  «веселье без функции — просто шум».
- Gen Z / Millennials предпочитают рецепт-вдохновение с интерактивом.

## Мост fun -> utility (что показывать после игры)
Принцип: ни один спин/квиз не заканчивается тупиком.

После колеса — экран результата с тремя действиями:
1. Готовить дома -> рецепт -> недостающее в Grocery/корзину.
2. Сканировать продукты -> что собрать из наличия.
3. Поесть вне дома -> рестораны по кухне / бронь шефа.

После квиза — подборка: 3-5 рецептов + 1 ресторан + опция «позвать шефа».
Квиз = мягкий онбординг предпочтений, заполняет профиль.

Правило перехода:
- fun-экран обязан иметь видимую кнопку «дальше в дело».
- Результат игры пишется в профиль -> улучшает Recommendations.
- Регистрация лениво (no sign-up = вовлечённость).

Метрика успеха моста: доля игровых сессий, доведённых до «серьёзного» действия
(открыт рецепт, добавлено в корзину, начата бронь).

## Что это значит для Le Chef
1. Ставка на «мост» fun -> utility как главную отстройку.
2. Не побеждать чистые базы рецептов объёмом.
3. Усиливать уникальное: поводы + готовка + рестораны + шеф.
