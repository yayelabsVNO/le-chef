## Главный вывод
Рынок чётко делится на два лагеря, и **почти никто не соединяет их в одном продукте**:
- «Серьёзные» утилиты: фото → ингредиенты → рецепты → список покупок (Pantry Pic, NomNomBear, SuperCook).
- «Весёлые» решалки нерешительности: колесо/квиз «что/где поесть» (SpinDine, ChooseMy.Food, Food-Decider).

**Пробел рынка, который закрывает Le Chef:** единый продукт, где fun-вход (колесо/квиз)
ведёт к серьёзному ядру (скан продуктов → рецепты → покупки → даже бронь шефа/ресторана).
Плюс почти ни у кого нет связки «домашняя готовка ↔ рестораны ↔ приглашённый шеф» в одном месте.

## Что подтверждает гипотезу «от fun к серьёзному»
- Кейс Food-Decider: «no sign-up» + колесо дали мгновенную вовлечённость; «the wheel thing is addictive».
- Кейс SpinDine: геймификация работает только если за ней стоит реальная система (спины/лимиты/тиры) — «fun without function is just noise».
- Статистика из обзоров: значимая доля Gen Z / Millennials предпочитает рецепт-вдохновение с интерактивом/геймификацией; решение про еду — эмоциональное и ситуативное, а не логическое.

## Риски отстройки
- SuperCook силён по базе рецептов — Le Chef не выиграет «количеством», нужно брать
  фото-сканом + поводами + ресторанами + игрой.
- Колесо/квиз — уже не уникальная фича сама по себе; уникальна **связка** игры и
  глубокого ядра внутри одного флоу.

## Что это значит для Le Chef
1. Делать ставку на «мост» fun → utility как на главную отстройку.
2. Не пытаться победить чистые базы рецептов объёмом.
3. Усиливать уникальное: поводы (business meeting / partner / friends) + готовка + рестораны + шеф.

## Мост fun → utility (что показывать после игры)

Главный принцип: **ни один спин/квиз не заканчивается тупиком**. После результата —
всегда переход в серьёзное ядро. Колесо и квиз — это вход, а не отдельная игрушка.

### После колеса (Spin the Wheel)
Выпадает блюдо/категория → экран результата с тремя действиями:
1. **Готовить дома** → рецепт → не хватает ингредиентов → добавить в Grocery/корзину.
2. **Сканировать продукты** → проверить, что из выпавшего можно собрать из наличия.
3. **Поесть вне дома** → рестораны по этой кухне на карте / бронь шефа.

### После квиза (Quiz)
Вопросы про настроение/повод/время/диету → на выходе НЕ один ответ, а персональная
подборка: 3–5 рецептов + 1 ресторан + опция «позвать шефа» (Occasion). Квиз =
мягкий онбординг предпочтений, который заполняет профиль для будущих рекомендаций.

### Правило перехода
- fun-экран обязан иметь видимую кнопку «дальше в дело» (рецепт / корзина / бронь).
- Результат игры пишется в профиль (история, вкусы) → улучшает Recommendations.
- Лёгкий вход без обязательной регистрации; регистрацию просить только при
  сохранении избранного/корзины/брони (по примеру Food-Decider: no sign-up = вовлечённость).

### Метрика успеха моста
Доля игровых сессий, которые довели до «серьёзного» действия (открыт рецепт,
добавлено в корзину, начата бронь). Если колесо крутят, но дальше не идут — мост не работает.
